Python is a great scripting language. However, sometimes we need to create larger python projects that need to be properly structured so that others can understand what's going as well as being able to distribute the project. Most packages you install via pip follow this structure.

Before Starting

One thing that I really need to make clear with regards to Python, is that you must user a version manager. This is because not only does Python have a major version divide when it comes to python 2 and 3, it also has other divides when it comes to minor versions. For example, Python 3.5 introduced the async keyword to make using the asyncio module easier. So, its best to know what version of Python you're using, and stick to that when you're working on your project.

I personally suggest using the pyenv project, and installing it through the pyenv installer. However, others have suggested that using pythonz. I personally thing that pyenv is a better choice, but you're free to choose between these two options.

Virtualenv

The virtualenv package creates a virtual environment for you with its own. From the wise words of the Python guide, this is why you should use virtualenv.

A Virtual Environment is a tool to keep the dependencies required by different projects in separate places, by creating virtual Python environments for them. It solves the “Project X depends on version 1.x but, Project Y needs 4.x” dilemma, and keeps your global site-packages directory clean and manageable.

Hitchhiker's Guide to Python

Installing virtualenv can be done through pip.

pip install virtualenv

Creating a virtualenv is rather simple.

virtualenv .venv

Activating the virtual environment is simple too.[1]

source .venv/bin/activate
